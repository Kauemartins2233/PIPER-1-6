O sistema de plug-ins do X-Plane é baseado em bibliotecas de vínculo dinâmico
(DLLs). O componente central desse sistema é o X-Plane Plugin Manager (XPLM) que
é uma biblioteca de código — compilada como uma DLL — responsável por gerenciar
os plug-ins. O X-Plane vincula-se ao XPLM e todos os plug-ins se conectam ao XPLM.
Desta forma, o X-Plane Plugin Manager atua como o hub central no sistema de plug-ins
( , 2018a).

Os plug-ins contêm o código que será executado dentro do simulador. Eles exportam

para alterar o simulador, ler dados, criar interface do usuário, etc. O XPLM exporta essas
funções para o diretório DLL para que o plug-in possa localizá-las e chamá-las (

6.3.3 Interface de Entrada e Saída de Dados
A comunicação entre o X-Plane e a plataforma de testes, neste trabalho, foi feita
através do plug-in X-Plane Pegasus desenvolvido para interfacear a entrada e saída de
dados do simulador de voo. Durante cada ciclo de voo da aeronave no simulador X-
Plane, esse plug-in é responsável por receber e decodificar o pacote de dados enviado pela
Unidade Central de Processamento (UCP) com as variáveis do vetor de entrada, codificar
e enviar para o Sistema de Navegação (SN) o pacote de dados com as variáveis do vetor
de saída da dinâmica de simulação da aeronave.
